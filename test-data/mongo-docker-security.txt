id="start-it-with-docker">Start it with docker</h2>
<p>The most simple way to get MongoDB instance in your machine is using docker to run <a target="_blank" href="https://hub.docker.com/_/mongo">mongodb image</a> on docker hub as below:</p>
<pre><code class="lang-bash">docker run --name mongodb -p 27017:27017 mongo
</code></pre>
<p>By specifying <code>-p 27017:27017</code>, you can access to it with the connection string such as <code>mongodb://localhost:27017</code> or <code>mongodb://0.0.0.0:27017</code>, as same as installed MongoDB server.</p>
<p>This database instance is only available locally. In the case you want to access to it from another computer, you have to overwrite the default config file with a custom MongoDB configuration file.</p>
<h2 id="allow-to-connect-from-outside">Allow to connect from outside</h2>
<p>The command looks like below:</p>
<pre><code class="lang-bash">docker run --name mongodb -p 27017:27017 -v /path/to/custom/mongod.conf:/etc/mongod.conf mongo
</code></pre>
<p>In which, the <code>mongod.conf</code> file has been customized to allow to connect from outside:</p>
<pre><code class="lang-conf"># mongod.conf

# other settings

# network interfaces
net:
  port: 27017
  # bindIp: 127.0.0.1  # this is default setting
  bindIp: 0.0.0.0  # this is what changed

# other settings
</code></pre>
<p>But since now your database is public and no longer secure.</p>
<p>In my experience, the first thing we should do in this situation is change the default port. </p>
<h2 id="change-default-port">Change default port</h2>
<p>Let's map it to another free port:</p>
<pre><code class="lang-bash">docker run --name mongodb -p 27117:27017 -v /path/to/custom/mongod.conf:/etc/mongod.conf mongo
</code></pre>
<p>So that the connection string will become <code>mongodb://IP_ADDRESS:27117</code>, not default port.</p>
<p>As you know, there are many bots those automatically scan for public databases on the internet. They try to check the popular default ports, such as 3306 (mysql), 5432 (posgres), 6379 (redis), 27017 (mongo), etc. Changing to a strange port may help you in almost cases.</p>
<p>However, that's not enough.</p>
<h2 id="enable-authentication">Enable authentication</h2>
<p>By default MongoDB will start with authentication disabled. We should enable it with a little bit change to entrypoint. In order to make things easier management, we create a docker-compose.yaml file as below:</p>
<pre><code class="lang-yaml">version: "3"

services:
  mongodb:
    image: mongo
    container_name: mongodb
    ports:
      - 27117:27017
    volumes:
      - ./mongod.conf:/etc/mongod.conf
    entrypoint: ["mongod", "--auth", "--config", "/etc/mongod.conf"]
</code></pre>
<p>In the above .yaml file, we added our custom entrypoint to run mongodb with <code>--auth</code> flag. Now when we run it with docker-compose, we should get MongoDB within a container named <code>mongodb</code> similar to the <code>docker run</code> command earlier.</p>
<p><img src="https://i.imgur.com/kXhOn6J.png" alt /></p>
<p>And the following is the most important parts of this article...</p>
<h3 id="create-users-with-specific-permissions">Create users with specific permissions</h3>
<p>Until now, our MongoDB instance does not have any user yet. We will get into its container and create a few ones.</p>
<p>Firstly, let's get the interactive mongo shell by running:</p>
<pre><code class="lang-bash">docker <span class="hljs-built_in">exec</span> -it mongodb mongo
</code></pre>
<p>With opening mongo shell, we can now run the commands to switch to <code>admin</code> database and create a user named <code>boss</code> with admin permission on any database:</p>
<pre><code class="lang-bash">use admin
db.createUser({
  user: <span class="hljs-string">'boss'</span>,
  <span class="hljs-built_in">pwd</span>: <span class="hljs-string">'YourVeryComplexPassword'</span>,
  roles:[<span class="hljs-string">'userAdminAnyDatabase'</span>]
})
</code></pre>
<p>Done, press Ctrl + C to exit the shell and leave that session.</p>
<p><img src="https://i.imgur.com/MayJG2g.png" alt /></p>
<p>Note that, at this time, we had already a MongoDB instance run with authentication mode enabled, and an admin user. The connection string for <code>boss</code> user now looks like:</p>
<p><code>mongodb://boss:YourVeryComplexPassword@IP_ADDR:27117</code></p>
<p>But that's power user, don't use it for regular applications. Instead, we should create more users for different purposes.</p>
<p>To create more user or database, we have to come back to mongo shell as <code>boss</code> user:</p>
<pre><code class="lang-bash">docker <span class="hljs-built_in">exec</span> -it mongodb mongo -u boss
</code></pre>
<p>Enter password when asked and you should see something like:</p>
<p><img src="https://i.imgur.com/maIXyvg.png" alt /></p>
<p>Assume that we have a plan to use a database <code>blogs</code> to store our great blog posts. We should add a user named <code>writer</code> which can write to this database, and a user named <code>reader</code> which can read only.</p>
<p>Do that as below:</p>
<pre><code># <span class="hljs-selector-tag">go</span> <span class="hljs-selector-tag">to</span> <span class="hljs-selector-tag">database</span> `<span class="hljs-selector-tag">admin</span>`
<span class="hljs-selector-tag">use</span> <span class="hljs-selector-tag">admin</span>

# <span class="hljs-selector-tag">add</span> <span class="hljs-selector-tag">writer</span>
<span class="hljs-selector-tag">db</span><span class="hljs-selector-class">.createUser</span>({
  <span class="hljs-attribute">user</span>: <span class="hljs-string">'writer'</span>,
  pwd: <span class="hljs-string">'YourComplexPassword'</span>,
  roles:[
    {
      role: <span class="hljs-string">'readWrite'</span>,
      db: <span class="hljs-string">'blogs'</span>
    }
  ]
})

# <span class="hljs-selector-tag">add</span> <span class="hljs-selector-tag">reader</span>
<span class="hljs-selector-tag">db</span><span class="hljs-selector-class">.createUser</span>({
  <span class="hljs-attribute">user</span>: <span class="hljs-string">'reader'</span>,
  pwd: <span class="hljs-string">'YourSimplePassword'</span>,
  roles:[
    {
      role: <span class="hljs-string">'read'</span>,
      db: <span class="hljs-string">'blogs'</span>
    }
  ]
})
</code></pre><p>Done, press Ctrl + C to exit the shell and leave that session again.</p>
<p><img src="https://i.imgur.com/gmHzXFL.png" alt /></p>
<p>Now we have 2 new users, one for read/write and other for read only action. Their connection strings are:</p>
<p><code>mongodb://writer:YourComplexPassword@IP_ADDR:27117</code>
<code>mongodb://reader:YourSimplePassword@IP_ADDR:27117</code></p>
<p>With the apps or API endpoints those only load and show blog posts, we just need to use the connection string for <code>reader</code>. Otherwise, when we need to add more blog posts, we will connect to database as <code>writer</code>.</p>
<p>That's it. Enjoy playing with docker and MongoDB :)</p>
